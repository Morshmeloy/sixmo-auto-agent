# Sixmo Auto Agent

[![Python](https://img.shields.io/badge/python-3.13-blue.svg)](https://www.python.org/)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

**Sixmo Auto Agent** — AI‑агент для полного автоматического прохождения тестовой формы [sixmo.ru](https://sixmo.ru/).  
Он работает с динамическими элементами, загружает файлы и отвечает на вопросы любой тематики, используя генеративные модели (GitHub Models).

Проект оформлен как **tool/skill** – простая функция `submit_form()`, которую можно вызывать из любого агентного фреймворка.

---

## Выполнение требований

### Автоматическое прохождение формы от начала до конца
Функция `submit_form()` запускает браузер, нажимает «Начать задание», последовательно обрабатывает все шаги, заполняет поля и извлекает финальный идентификатор.

### Загрузка файла на втором этапе
Скрипт определяет поле типа `file`, запрашивает у LLM подходящее слово (например, «волшебство»), создаёт временный текстовый файл и загружает его.

### Устойчивость к нестабильности интерфейса
Поля находятся не по жёстким селекторам, а по ролям (`input`, `select`, `textarea`, `[role="textbox"]` и др.). Вопросы извлекаются из окружающего DOM (метки, родительские блоки, предыдущие элементы). Форма может менять порядок полей и этапов – скрипт адаптируется к любой структуре.

### Оформление как tool/skill
Вся логика инкапсулирована в функцию `submit_form(headless: bool = False) -> str`. Она может быть импортирована и вызвана любым агентом (OpenAI Assistant, LangChain, OpenClaw и др.). Возвращаемое значение – идентификатор завершённой формы.

## Быстрый старт

### Локальный запуск

1. Клонируйте репозиторий
   ```bash
   git clone https://github.com/ваш-логин/sixmo-auto-agent.git
   cd sixmo-auto-agent
   ```

2. Установите зависимости
   ```bash
   pip install -r requirements.txt
   playwright install chromium
   ```

3. Настройте токен GitHub  

   Создайте файл `.env` в корне проекта:
   ```env
   GITHUB_TOKEN=ваш_токен_здесь
   ```

   Токен можно получить в настройках GitHub → Developer settings → Personal access tokens → Tokens (classic).  
   Токен бесплатный и нужен только для доступа к GitHub Models.

4. Запустите агента
   ```bash
   python agent.py
   ```

   По умолчанию браузер работает в режиме с графическим интерфейсом (`headless=False`).  
   Для фонового режима измените в конце файла `submit_form(headless=True)`.

### Запуск в Docker

```bash
docker build -t sixmo-auto-agent .
docker run --env-file .env sixmo-auto-agent
```

Контейнер автоматически использует `headless=True` и не требует X‑сервера.

## Как это работает

1. **Запуск браузера** – Playwright запускает Chromium с аргументами, скрывающими автоматизацию.
2. **Анализ страницы** – на каждом шаге собираются все видимые поля ввода.
3. **Извлечение вопроса** – для каждого поля определяется текст вопроса (через `label`, родительские блоки или предыдущий элемент).
4. **Получение вариантов** – для `<select>` и радиогрупп извлекаются доступные опции.
5. **Генерация ответа** – формируется промпт, отправляемый в GitHub Models (`gpt-4o-mini`). Модель возвращает короткий ответ или описание файла.
6. **Заполнение полей** – ответ вводится в текстовые поля, выбираются опции, загружается файл.
7. **Навигация** – нажимается кнопка перехода («Продолжить», «Далее», «Зафиксировать идентификатор»).
8. **Извлечение идентификатора** – после завершения формы из страницы извлекается 12‑значный шестнадцатеричный идентификатор.

## Настройка

- **Модель** – измените `self.model_name` в `__init__` на любую модель, доступную через GitHub Models (например, `gpt-4o`, `gpt-4o-mini`).
- **Язык интерфейса** – если форма на другом языке, отредактируйте список `possible_texts` в методе `click_next_button()`.
- **Режим работы** – передайте `headless=True` в `submit_form()` для работы без отображения браузера.
